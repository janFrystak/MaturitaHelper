1 Sestavování počítače – komponenty a parametry

Základní deska
Motherboard – páteř systému, propojuje všechny komponenty
Obsahuje: socket CPU, sloty RAM, PCIe sloty, chipset, BIOS/UEFI čip, konektory SATA/M.2, I/O porty
Parametry: socket (kompatibilita s CPU), form factor (ATX / mATX / ITX), podporované frekvence RAM, počet PCIe slotů
CPU

Parametry: počet jader a vláken · taktovací frekvence (GHz, base/boost) · TDP (W) · cache L1/L2/L3 · socket · architektura (x86-64, ARM)
RAM
Operační paměť – dočasná paměť pro běžící programy a OS
Typy: DDR4, DDR5 (aktuální standard)  ·  Moduly: DIMM (desktop), SO-DIMM (notebook)
Parametry: kapacita (GB) · frekvence (MHz) · latence (CL timing) · dual-channel
GPU

Typy: dedikovaná (diskrétní, vlastní VRAM) vs. integrovaná (součást CPU, sdílí RAM)
Parametry: VRAM (GB) · paměťová šířka pásma · TDP · rozhraní PCIe x16
Úložiště
HDD nebo SSD – trvalé uložení dat (viz sekce 4)
Rozhraní: SATA III (~550 MB/s) · M.2 NVMe PCIe (až ~7 000 MB/s)
PSU
Zdroj – napájí celý systém
Parametry: výkon (W) · účinnost – 80 Plus certifikace (Bronze / Gold / Platinum) · modulárnost
Chlazení
Vzduchové (heatsink + ventilátor) nebo vodní (AIO / custom loop)
Parametry: TDP rating · hlučnost (dB) · výška chladiče (kompatibilita s case)
Case
Form factor (ATX / mATX / ITX) · airflow design · počet drive slotů · kabelmanagement

Registry

Malé, extrémně rychlé úložiště přímo uvnitř procesoru. Jsou součástí čipu – přístup bez latence paměťového systému. Velikost odpovídá architektuře (32-bit → 32-bit registry, 64-bit → 64-bit registry).

Bázový/indexový registrAdresování paměti při práci s poli a strukturamiRBP, RSI, RDI

Instrukce a instrukční sada (ISA)

Instrukce = základní příkaz, který CPU umí přímo vykonat.
Skládá se z: operační kód (opcode) + operandy (adresa nebo data)

Příklady: MOV (přesun dat)  ·  ADD / SUB / MUL / DIV  ·  JMP (skok)  ·  CMP (porovnání)  ·  PUSH / POP

CISC – Complex Instruction Set

Velké množství složitých instrukcí, proměnná délka instrukce

Příklady: x86, x86-64 (Intel, AMD)

RISC – Reduced Instruction Set

Malá sada jednoduchých instrukcí, pevná délka, rychlejší dekódování

Příklady: ARM, RISC-V, Apple M-series

Cyklus CPU – Fetch → Decode → Execute → Writeback

  1. Fetch – načtení instrukce z paměti na adrese uložené v PC (Program Counter)
  2. Decode – dekodér zjistí, o jakou instrukci jde a co potřebuje
  3. Execute – ALU nebo FPU instrukci provede
  4. Writeback – výsledek zapsán zpět do registru nebo paměti

Vnitřní paměti (primární)

RAM – Random Access Memory

Volatilní – data se ztrácejí po vypnutí napájení. Slouží jako pracovní prostor CPU pro právě běžící programy a OS.

DRAM – Dynamic RAM
  • Každý bit uložen v kondenzátoru – drží náboj jen krátce
  • Nutný periodický refresh (obnova náboje) – stovky ms
  • Jednoduchá struktura → levná a hustá, vysoké kapacity
  • Pomalejší přístupová doba (~60–100 ns)
  • Použití: operační paměť (RAM moduly)
SRAM – Static RAM
  • Každý bit uložen v klopném obvodu (bistabilní hradla)
  • Data drží dokud je napájení – žádný refresh
  • Složitá struktura (6 tranzistorů/bit) → drahá, malá kapacita
  • Extrémně rychlá přístupová doba (~1–5 ns)
  • Použití: cache L1/L2/L3 v CPU

Moduly RAM

DIMM (Desktop)

Dual Inline Memory Module · 288 pinů · 64-bit sběrnice · standardní pro stolní počítače

SO-DIMM (Notebook)

Small Outline DIMM · fyzicky kratší (67,6 mm) · 260 pinů · notebooky, NUC, mini-PC

Vnější paměti (sekundární)

Nevolatilní, trvalé uložení dat. Jsou pomalejší než RAM, ale kapacitně mnohem větší. Patří sem: HDD, SSD, optická média, magnetické pásky, USB flash disky, SD karty – viz sekce 4 a 6.

4 Pevný disk – princip funkce, HDD vs. SSD

HDD – Hard Disk Drive (magnetický pevný disk)

Princip funkce
  • Data ukládána magneticky na rotující plotny (platter) pokryté ferromagnetickým materiálem
  • Čtecí/zapisovací hlavičky plovoucí těsně nad povrchem mění a čtou magnetizaci
  • Plotny rotují na vřetenu: 5 400 / 7 200 / 10 000 / 15 000 RPM
  • Přístup k datům = mechanický pohyb → latence (seek time ~5–10 ms)
Struktura dat na plotně
  • Stopa (track) – soustředná kružnice na plotně
  • Sektor – nejmenší adresovatelná jednotka (512 B nebo 4 096 B = 4K)
  • Cylindr – souhrn stejných stop na všech plotnách
  • Cluster – skupina sektorů, základní alokační jednotka souborového systému

Nevýhody HDD: Pomalý náhodný přístup · pohyblivé části = riziko poruchy při pádu · hlučný · vyšší spotřeba energie

SSD – Solid State Drive (flash úložiště)

Princip funkce
  • Data ukládána v NAND Flash paměťových buňkách
  • Každá buňka ukládá elektrický náboj v plovoucím hradlu tranzistoru
  • Žádné pohyblivé části → odolnost, ticho, nízká latence
  • Řídí řadič (controller) + DRAM cache (u kvalitnějších SSD)
Rozhraní SSD
  • SATA III – starší, ~550 MB/s, fyzický konektor SATA
  • M.2 SATA – fyzicky M.2 slot, ale rychlostí SATA
  • M.2 NVMe – PCIe linka, až ~7 000 MB/s (PCIe 4.0), nízká latence

Parametry: sekv. čtení/zápis (MB/s) · TBW (životnost v TB zapsaných) · DRAM cache ano/ne

5 Vyrovnávací paměti (Cache)

Cache = rychlá SRAM paměť mezi pomalejší pamětí a procesorem. Uchovává kopii naposledy / nejčastěji používaných dat → snižuje průměrnou dobu přístupu.

Principy lokality

Prostorová lokalita

Pokud přistoupíme k adrese X, brzy přistoupíme i k adresám blízkým X (pole, sekvenční instrukce)

Časová lokalita

Pokud přistoupíme k adrese X, brzy k ní přistoupíme znovu (cykly, opakované proměnné)

Cache HIT ✓Požadovaná data nalezena v cache → rychlý přístup bez čekání na RAM
Cache MISS ✗Data nejsou v cache → nutno načíst z pomalejší paměti (penalizace latencí)

L2256 KB – 2 MB~10–20 cyklůNa čipu, uvnitř jádraPer-core
L34–64 MB~30–50 cyklůNa čipu, mimo jádraSdílená všemi jádry

Politiky zápisu

Write-throughData zapsána současně do cache i do RAM · konzistentní stav · pomalejší
Write-backData zapsána nejprve jen do cache, do RAM až při uvolnění bloku · rychlejší · riziko ztráty při výpadku

Cache mimo CPU

  • Disk cache / Buffer – RAM buffer pevného disku (8–256 MB); dočasně ukládá čtená/zapisovaná data
  • TLB (Translation Lookaside Buffer) – speciální cache pro překlad virtuálních adres na fyzické

6 Paměťová média – rozdělení, vlastnosti, příklady

Podle principu záznamu

Magnetická média

Princip: magnetizace ferromagnetického povrchu

Příklady: HDD · magnetické pásky LTO · diskety (historicky)

+ Vysoká kapacita, nízká cena/GB
− Citlivá na magnetické pole, pohyblivé části

Flash paměti (elektronická)

Princip: NAND flash buňky, elektrický náboj

Příklady: SSD · USB flash disk · SD/microSD karta · eMMC · UFS

+ Rychlá, tichá, odolná
− Omezený počet zápisů

Optická média

Princip: laser čte/zapisuje do reflexní vrstvy (změna fáze nebo barviva)

CD 700 MB · DVD 4,7 GB (DL: 8,5 GB) · Blu-ray 25–100 GB

Typy: ROM (jen čtení) · -R (jednorázový zápis) · -RW (přepisovatelný)

Polovodičová paměť (RAM – volatilní)

Princip: DRAM kondenzátory nebo SRAM klopné obvody

Viz sekce 3 – slouží jako operační paměť, ne jako trvalé úložiště

Podle přístupu k datům

Sekvenční přístupData čtena v pořadí – nelze skočit přímo na libovolnou pozici
Příklad: magnetické pásky (LTO)
Přímý (náhodný) přístupPřímý skok na libovolnou adresu bez průchodu předchozími daty
Příklady: RAM, SSD, HDD

Podle volatility

VolatilníData se ztrácejí po odpojení napájení → RAM (DRAM, SRAM)
NevolatilníData přetrvávají bez napájení → Flash, HDD, optická média, ROM
