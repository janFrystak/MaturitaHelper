1 Co je OOP

Objektově orientované programování (OOP) = přístup k programování, kde se program skládá z objektů. Objekty představují reálné věci nebo abstraktní koncepty (auto, student, účet). Cílem je přiblížit programování reálnému světu a usnadnit správu kódu.

Každý objekt má
  • Stav — určený atributy (proměnné uvnitř třídy)
  • Chování — určené metodami (funkce uvnitř třídy)
Výhody OOP
  • Lepší organizace kódu
  • Znovupoužitelnost
  • Snadnější údržba a rozšiřování
  • Přehlednost

2 Rysy OOP

Zapouzdření (Encapsulation)

Skrytí vnitřní implementace objektu před vnějším světem. Atributy bývají private a přistupuje se k nim přes gettery a settery. Zajišťuje bezpečnost dat a jasně určuje, jak se má objekt používat.

public class Student {
    private String jmeno;

    public String getJmeno() { return jmeno; }
    public void setJmeno(String jmeno) { this.jmeno = jmeno; }
}

Dědičnost (Inheritance)

Umožňuje, aby jedna třída (potomek) přebírala vlastnosti a metody jiné třídy (rodič). Šetří kód — stejné metody nepíšeme vícekrát. V Javě klíčové slovo extends. Třída může mít jen jednoho předka.

class Zvire {
    void jez() { System.out.println("Jím..."); }
}
class Pes extends Zvire {
    void stekej() { System.out.println("Haf!"); }
}

Polymorfismus (Polymorphism)

Různé objekty reagují na stejnou zprávu různě. Jedno rozhraní, více implementací. Dvě formy:

Přetěžování (Overloading) Stejná metoda, jiný počet nebo typ parametrů — v rámci jedné třídy. 
void vypis(int a) {}
void vypis(String a) {}
Přepisování (Overriding) Potomek přepíše metodu rodiče svým chováním. Anotace @Override. 
class Pes extends Zvire {
    @Override
    void zvuk() {
        System.out.println("Haf!");
    }
}

Abstrakce (Abstraction)

Odděluje důležité vlastnosti od nedůležitých detailů. U třídy Auto nás zajímá, že má metodu jed(), ale nezajímá nás, jak přesně motor funguje. V Javě: abstraktní třídy a rozhraní.

public abstract class Auto {
    public abstract void jed();  // bez těla — potomek musí implementovat
}
public class Skoda extends Auto {
    public void jed() { /* implementace */ }
}

3 Rozhraní (Interface) a implementace

Rozhraní = smlouva, která říká, jaké metody musí třída implementovat. Neobsahuje kód metod, jen jejich hlavičky. Klíčové slovo implements. Třída může implementovat kolik rozhraní chce.

interface Zvuk {
    void hraj();
}
class Kytara implements Zvuk {
    public void hraj() { System.out.println("Brnknutí!"); }
}

4 Pojmy: třída, objekt, instance, atributy, metody, konstruktor

PojemDefinice
Třída (class)Šablona (vzor), podle které se vytváří objekty. Definuje atributy a metody.
Objekt (object)Konkrétní realizace třídy, existuje v paměti (heap).
InstanceJiný název pro objekt vytvořený z třídy. Auto a1 = new Auto("Škoda"); → a1 je instance třídy Auto.
AtributyProměnné uložené ve třídě — tvoří stav objektu.
MetodyFunkce definované ve třídě — tvoří chování objektu.
KonstruktorSpeciální metoda volaná při vytváření objektu (new). Obvykle inicializuje atributy. Má stejný název jako třída, nemá návratový typ.
public class Auto {
    private String znacka;
    private int rychlost;

    // konstruktor
    public Auto(String znacka) {
        this.znacka = znacka;
        this.rychlost = 0;
    }
    public void zrychli(int oKolik) { rychlost += oKolik; }
}

// vytvoření instance:
Auto a1 = new Auto("Škoda");  // new → objekt v paměti (heap)
a1.zrychli(50);               // a1 obsahuje referenci na objekt

5 Datové typy a jejich dělení

Primitivní typy Jednoduché datové typy. Ukládají se přímo jako hodnota.
int double char boolean long float
Referenční typy Odkazy na objekty. V paměti je uložen odkaz (reference), ne hodnota samotná.
String ArrayList vlastní třídy
⚠ Pozor: porovnávání Stringů

== porovnává u objektů reference, ne obsah. Pro porovnání hodnoty použij .equals().

String a = "Ahoj";
String b = "Ahoj";
if (a == b) { }          // ❌ chyba — porovnává reference
if (a.equals(b)) { }     // ✅ správně — porovnává obsah

Třídní vs. instanční atributy

Instanční atribut Patří konkrétnímu objektu. Každá instance má svoji vlastní hodnotu.
Třídní (statický) atribut Patří celé třídě, sdílený všemi instancemi. Deklaruje se pomocí static. 
public class Osoba {
    public static int pocetLidi = 0;  // třídní — sdílený
    private String jmeno;             // instanční — každý má vlastní

    public Osoba(String jmeno) {
        this.jmeno = jmeno;
        pocetLidi++;
    }
}

Klíčová slova

thisOdkaz na aktuální objekt. Rozlišuje parametr od atributu: this.jmeno = jmeno
superOdkaz na rodiče. Volání konstruktoru nebo metody rodiče: super.metoda()
finalProměnná → nelze změnit · Metoda → nelze přepsat (override) · Třída → nelze dědit

7 Konvence pro zápis kódu a čistý kód

Konvence pojmenování
  • Třídy: PascalCase — Auto, PovolaniZahradnik
  • Metody a proměnné: camelCase — getJmeno(), vekStudentu
  • Konstanty: VELKYMI_PISMENY — MAX_RYCHLOST
  • Balíčky: malá písmena — cz.skola.projekt
Zásady čistého kódu
  • Jeden úkol = jedna metoda (krátké metody)
  • Výstižné názvy — kód je dokumentace
  • Správné odsazení a prázdné řádky
  • Komentáře tam, kde kód není srozumitelný sám
  • DRY — Don't Repeat Yourself
