1 Základní charakteristika Javy a JDK/JRE/JVM

Java je objektově orientovaný, staticky typovaný programovací jazyk. Zdrojový kód (.java) se kompiluje do bytecode (.class), který spouští JVM.

JDK — Java Development Kit Nástroje pro vývoj: kompilátor javac, debugger, dokumentační nástroje. Obsahuje JRE.
JRE — Java Runtime Environment Prostředí pro běh programu. Obsahuje JVM + standardní knihovny. Stačí pro spuštění, ne pro vývoj.
JVM — Java Virtual Machine Spouští bytecode. Zajišťuje přenositelnost — stejný bytecode běží na Windows, Linux, macOS.
Bytecode

Mezikód v .class souborech. Nezávislý na OS — JVM ho přeloží do strojového kódu konkrétního procesoru. Princip: Write Once, Run Anywhere.

Vlastnosti Javy

PřenositelnostBytecode běží na libovolném OS se správným JVM
BezpečnostSandboxing, typová kontrola, žádné přímé přístupy do paměti
Garbage CollectorAutomatická správa paměti — uvolňuje objekty, které už nejsou používány

2 Kontejnery a pole (Array)

Kontejner = třída umožňující ukládat více objektů. Většina kontejnerů je v balíčku java.util. Výhody: dynamická velikost, hotové metody, práce s objekty.

ArrayList

Dynamické pole — mění velikost automaticky. Ukládá objekty (ne primitivní typy — používají se wrappery: Integer, Double…). Zachovává pořadí prvků.

Základní metody
  • add(prvek) — přidání na konec
  • add(index, prvek) — přidání na pozici
  • remove(index) — odstranění
  • get(index) — přístup k prvku
  • size() — počet prvků
  • contains(prvek) — obsahuje?
Výhody / Nevýhody
  • ✅ Jednoduché použití
  • ✅ Dynamická velikost
  • ✅ Hotové metody (třídění, hledání)
  • ❌ Pomalejší při vkládání doprostřed
  • ❌ Pouze objekty (ne primitiva přímo)
ArrayList<String> list = new ArrayList<>();
list.add("Ahoj");
list.add("Světe");
System.out.println(list.get(0)); // vypíše: Ahoj
System.out.println(list.size()); // vypíše: 2

Pole (Array)

Ukládá více hodnot stejného typu. Má pevnou velikost — nelze po vytvoření změnit. Přístup přes index (od 0).

int[] pole = {1, 2, 3, 4, 5};
System.out.println(pole[0]);  // vypíše: 1
System.out.println(pole.length); // délka pole

// for-each iterace:
for (int x : pole) {
    System.out.println(x);
}

3 Cykly a větvení

Cykly

for Když známe počet opakování. Řídící proměnná, podmínka, krok. 
for (int i = 0; i < 5; i++) {
    System.out.println(i);
}
while Opakuje se, dokud platí podmínka. Podmínka se kontroluje před každým průchodem. 
int i = 0;
while (i < 5) {
    System.out.println(i);
    i++;
}
do-while Provede se alespoň jednou. Podmínka se kontroluje až po průchodu. 
do {
    System.out.println(i);
    i++;
} while (i < 5);

Větvení

if / else if / else Základní podmíněné větvení. 
if (a > 0) {
    System.out.println("Kladné");
} else if (a == 0) {
    System.out.println("Nula");
} else {
    System.out.println("Záporné");
}
switch Více možností podle hodnoty proměnné. 
switch (den) {
    case 1: System.out.println("Po"); break;
    case 2: System.out.println("Út"); break;
    default: System.out.println("Jiný");
}

5 Výjimky a ošetření chyb

Výjimky = mechanismus pro řešení chyb za běhu programu. Zabraňují pádu aplikace při neočekávaných situacích.

Checked výjimky Musí být ošetřeny nebo deklarovány (throws). Kompilátor je vyžaduje. Příklady: IOException, FileNotFoundException, SQLException
Unchecked výjimky (Runtime) Ošetření není povinné. Příklady: ArithmeticException (dělení nulou), NullPointerException, ArrayIndexOutOfBoundsException

Try-catch-finally

try {
    int x = 5 / 0;  // vyvolá ArithmeticException
} catch (ArithmeticException e) {
    System.out.println("Chyba: " + e.getMessage());
} catch (Exception e) {
    System.out.println("Obecná chyba");  // zachytí ostatní
} finally {
    System.out.println("Toto se provede vždy");
}
tryKód, kde může nastat chyba
catchZachycení konkrétní výjimky — lze mít více bloků catch
finallyProvede se vždy — i při výjimce. Vhodné pro uzavření souborů/spojení.

Listener (posluchač událostí)

Reaguje na akce uživatele — kliknutí, změna textu. Implementuje interface jako ActionListener. Dnes nejčastěji jako lambda výraz:

JFrame frame = new JFrame("Moje okno");
frame.setSize(400, 300);
frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);

JButton btn = new JButton("Klikni");
btn.addActionListener(e -> {
    System.out.println("Klik!");
});

frame.add(btn);
frame.setVisible(true);

7 Práce se soubory

Zápis do souboru Třída PrintWriter obalená BufferedWriter a FileWriter. Buffering = méně zápisů na disk → rychlejší. 
PrintWriter pw = new PrintWriter(
    new BufferedWriter(
        new FileWriter("soubor.txt")));
pw.println("Řádek textu");
pw.close(); // vždy zavřít!
Čtení ze souboru Třída Scanner nebo BufferedReader. 
Scanner sc = new Scanner(
    new BufferedReader(
        new FileReader("soubor.txt")));
while (sc.hasNextLine()) {
    String radek = sc.nextLine();
    System.out.println(radek);
}
sc.close();

Důležité: Práce se soubory vyvolává checked výjimky (IOException) — vždy obalit do try-catch nebo deklarovat throws IOException. Vždy soubor po použití close()!

8 Git a GitHub

Git Systém pro správu verzí zdrojového kódu — funguje lokálně. Sleduje historii změn, umožňuje návrat k libovolné verzi, větvení (branches), slučování (merge).
GitHub Internetová služba pro sdílení Git repositářů — funguje v cloudu. Spolupráce více vývojářů, pull requesty, code review, CI/CD.
