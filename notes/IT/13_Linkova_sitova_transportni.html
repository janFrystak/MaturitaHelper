1 Typy přenosu dat – datagram, okruh, virtuální kanál

Datagramové sítě (přepínání paketů)

Data se rozdělí na pakety. Každý paket nese adresu odesílatele i příjemce a jde sítí samostatně — může zvolit jinou cestu.

  • ✓ Efektivní využití linek
  • ✓ Odolnost proti výpadkům
  • ✗ Pakety mohou dorazit v jiném pořadí nebo se ztratit
  • ✗ Výpočetně náročné směrování každého paketu
Spojově orientované sítě (přepínání okruhů)

Nejdřív se naváže trasa, pak se posílají data, pak se spojení ukončí. Celá linka je vyhrazena jednomu spojení. Historicky: PSTN (klasická telefonní síť).

  • ✓ Zaručená kapacita, stálá latence
  • ✗ Neefektivní využití linek (linka čeká i když nikdo nemluví)
Sítě s virtuálními kanály

Kombinace obou přístupů. První paket nese plnou adresu + identifikátor spojení. Směrovače si spojení zapamatují. Další pakety nesou jen ID.

  • ✓ Jednodušší směrování (stačí ID)
  • ✓ Pakety nedorazí v nesprávném pořadí
  • ✗ Při výpadku se spojení rozpadne

2 Směrování paketů

Směrování = rozhodování, kudy paket půjde sítí. Provádí ho router: přijme paket → přečte cílovou IP → podívá se do směrovací tabulky → pošle dál.

Směrovací tabulka a hledání next-hop

Obsah směrovací tabulky
  • Cílová síť — IP adresa sítě a maska
  • Next hop — IP adresa dalšího routeru
  • Default gateway — kam poslat, pokud nic nevyhovuje
Algoritmus hledání
  1. Projde tabulku od prvního řádku
  2. Pokud cílová IP patří do sítě na řádku → pošle na next-hop a skončí
  3. Pokud ne → přejde na další řádek
  4. Pokud nic nevyhovuje → použije default gateway, nebo zahodí

Směrovací protokoly

RIP — Routing Information Protocol Starší, jednoduchý. Routery si posílají celé směrovací tabulky. Vybere nejkratší cestu dle hop count. Omezení: max. 15 skoků — nevhodný pro velké sítě. Pomalá aktualizace při výpadku.
OSPF — Open Shortest Path First Moderní, pro velké sítě. Buduje model celé sítě. Počítá nejlepší cestu algoritmem (Dijkstra). Výpočetně náročnější, ale rychle reaguje na změny.
BGP — Border Gateway Protocol Pro směrování mezi autonomními systémy (AS) — velcí operátoři, mezistátní linky. Páteřní protokol internetu.

3 Protokoly síťové vrstvy – IP, ICMP, ARP

IP — Internet Protocol

Co dělá IP
  • Adresuje zařízení — přiděluje logické IP adresy
  • Přenáší pakety mezi různými sítěmi (síťová vrstva = L3)
  • Fragmentuje pakety, pokud jsou příliš velké pro danou linku (MTU)
Vlastnosti IP
  • Bez spojení (connectionless) — každý paket je nezávislý
  • Nespolehlivý — nezaručuje doručení, pořadí ani integritu dat
  • Best-effort — udělá co může, ale bez garance
  • Spolehlivost zajišťuje až TCP nad IP
IPv4 32bitová adresa, zapsaná jako 4 desítková čísla (např. 192.168.1.1). Přibližně 4 miliardy adres — vyčerpáno. Řeší ho NAT a přechod na IPv6.
IPv6 128bitová adresa, zapsaná hexadecimálně (např. 2001:db8::1). 340 undeciliónů adres — prakticky nevyčerpatelné. Jednodušší hlavička než IPv4.

ICMP — Internet Control Message Protocol

Servisní protokol nad IP. Slouží k informování o nestandardních stavech a diagnostice sítě. Není určen pro přenos uživatelských dat.

  • ping — ověření dostupnosti hosta (Echo Request / Echo Reply)
  • traceroute/tracert — sledování trasy paketu přes sítě
  • TTL expired — informace, že paket byl zahozen (vypršel TTL)
  • Destination unreachable — cíl nedosažitelný

ARP — Address Resolution Protocol

Převádí IP adresu → MAC adresu. Síť potřebuje obojí: IP (logická, L3) pro směrování, MAC (fyzická, L2) pro přenos v rámci lokální sítě.

  1. Zařízení vyšle broadcast do celé sítě: „Kdo má IP 192.168.1.5?"
  2. Cílové zařízení odpoví svou MAC adresou (unicast)
  3. Výsledek se uloží do ARP cache — příště se neptáme znovu

IPv6 ekvivalent: ND (Neighbor Discovery)

Stavové vs. bezestavové protokoly

Bezestavový (stateless) Server řeší každý požadavek zcela nezávisle. Ihned po odpovědi může zapomenout vše. Jednoduché, škálovatelné.
HTTPDNS
Stavový (stateful) Výsledek závisí na předchozích požadavcích. Server si pamatuje stav spojení. Složitější, ale nutné pro přenos souborů.
FTPTCP

4 Číslo portu, socket a protokoly TCP / UDP

Port Číslo (0–65535) identifikující konkrétní službu nebo aplikaci na zařízení. Umožňuje multiplexování — více aplikací sdílí jednu IP adresu.
  • Porty 0–1023 = dobře známé (HTTP 80, HTTPS 443, FTP 21, SSH 22, DNS 53, SMTP 25)
  • Porty 1024–49151 = registrované
  • Porty 49152–65535 = dynamické/klientské
Socket Kombinace IP adresa + port. Jednoznačně identifikuje jedno konec spojení. Celé TCP spojení = dvojice socketů (odesílatel i příjemce).

192.168.1.5:54321 ↔ 93.184.216.34:443

TCP = spolehlivost

Three-way handshake: SYN → SYN-ACK → ACK. Každý segment musí být potvrzen (ACK). Ztracené segmenty jsou znovu odeslány. Řídí tok dat (flow control).

UDP = rychlost

Žádný handshake. Data ihned odeslána bez čekání na potvrzení. Poškozené datagramy zahozeny bez náhrady. Pořadí doručení není garantováno.

5 Detekce a oprava chyb

Při přenosu dat dochází k chybám vlivem: rušení · šumu · ztráty signálu. Cílem detekce je upravit data tak, aby příjemce poznal, zda k poškození došlo.

Princip detekce chyb

1. Dohoda na mechanismuOdesílatel a příjemce se předem dohodnou na kontrolním algoritmu (stanoven protokolem)
2. Připojení kontrolních datOdesílatel k datům připojí kontrolní posloupnost bitů (checksum, CRC…) vypočítanou z dat
3. Ověření příjemcemPříjemce přepočítá kontrolní hodnotu. Souhlasí → data pravděpodobně OK. Nesouhlasí → chyba jistá.
⚠ Žádný mechanismus nemůže odhalit všechny chyby!

Pokud kontrolní hodnota nesouhlasí → jistě došlo k chybě. Pokud souhlasí → data jsou pravděpodobně v pořádku, ale existuje malá pravděpodobnost, že chyba změnila data i checksum na jiné platné hodnoty. Lze pouze minimalizovat pravděpodobnost neodhalené chyby.

Metody detekce a opravy chyb

MetodaPrincipDetekceOpravaPoužití
Parita (CRC1) Přidá 1 bit tak, aby počet jedniček byl sudý (nebo lichý). Nejjednodušší CRC. ✅ Liché počty chyb ❌ Některé typy RAM
CRC (Cyclic Redundancy Check) Výsledek dělení polynomem. CRC32 v Ethernetu. Velká skupina kódů s podobnými vlastnostmi. ✅ Silná ❌ Ethernet, ZIP, PNG
Hammingovy kódy Přidává více paritních bitů na různých pozicích — lze lokalizovat chybný bit. ✅ ✅ Umožňuje opravit 1 bit ECC RAM, kosmické systémy
Kontrolní součet (checksum) Součet všech bajtů dat modulo 2^n. Jednoduchý výpočet. ✅ Slabší ❌ TCP, UDP, IP hlavička

Parita — detailní příklad

Data: 1011001 (4 jedničky = sudý počet). Přidáme paritní bit 0 → 10110010. Pokud se přenese jako 10010010 — příjemce spočítá jedničky, výsledek je lichý → chyba odhalena. Pokud se změní 2 bity → parita vyjde sudě → chyba neodhalena.

Kde se kontrola chyb provádí

Linková vrstva (L2)Detekce chyb — CRC v Ethernet rámci. Poškozený rámec se zahodí, ale nezajistí znovuodeslání.
Transportní vrstva (L4) — TCPDetekce + oprava: checksum + sekvenční čísla + ACK + retransmission. Čím vyšší vrstva, tím komplexnější kontrola.

7 Řízení přístupu ke sdílenému médiu – CSMA/CD

Problém: více zařízení chce vysílat současně po jednom médiu → dochází ke kolizím. Řeší to CSMA/CD — protokol Ethernetu (dnes přežitý díky switchům).

CSMA/CD — Carrier Sense Multiple Access with Collision Detection
  1. Zařízení poslouchá médium (Carrier Sense) — je volné?
  2. Pokud volné → začne vysílat, ale zároveň stále poslouchá
  3. Pokud detekuje kolizi (Collision Detection) → zastaví vysílání, vyšle jam signal
  4. Čeká náhodný čas (Binary Exponential Backoff) → zkusí znovu

Náhodný čas čekání → aby obě zařízení nezačala znovu přesně ve stejnou chvíli → snížení pravděpodobnosti opakované kolize.

CSMA/CA (Wireless)Collision Avoidance — Wi-Fi. Nelze detekovat kolize za přenosu → předchází jim čekáním a rezervací média (RTS/CTS).
Moderní Ethernet se switchemFull-duplex, dedikované segmenty pro každé zařízení → kolize prakticky nevznikají. CSMA/CD se dnes téměř nepoužívá.
